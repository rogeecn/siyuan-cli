# GitHub Repository Description Design

**Goal:** Add a concise bilingual GitHub repository description that positions the project as a practical, production-usable SiYuan CLI.

**Scope:** This design covers the remote GitHub repository description only. It does not change README wording or code.

## Requirements
- English first, Chinese second
- Same meaning in both languages
- Short enough to fit a GitHub repository description naturally
- Emphasize practical usage, publishing, and automation

## Recommended Description
`Human-friendly CLI for SiYuan Note with practical publishing and automation workflows. 面向思源笔记的易用命令行工具，支持实用的发布与自动化工作流。`

## Reasoning
- "Human-friendly CLI" matches the existing project identity in README
- "practical publishing and automation workflows" reflects the actual validated capabilities
- Chinese sentence mirrors the same meaning directly instead of becoming more promotional or more technical than the English half

## Non-Goals
- No homepage URL change
- No repository topics/tags change
- No README changes in this task
